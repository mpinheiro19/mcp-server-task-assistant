# Contributing to MCP - Development Lifecycle Manager

Thank you for your interest in contributing! We welcome all kinds of contributions: bug reports, feature requests, documentation improvements, and code.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Describe your changes** clearly in your pull request.
3. **Test your code** before submitting. Run all tests with `pytest`.
4. **Follow code style**: Use [PEP8](https://www.python.org/dev/peps/pep-0008/) and type hints where possible.
5. **Keep commits atomic** and write clear commit messages.

## Reporting Issues

- Use the GitHub Issues tab.
- Provide steps to reproduce, expected and actual behavior, and environment details.

## Code of Conduct

By participating, you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions?

Open an issue or start a discussion in the repository.
